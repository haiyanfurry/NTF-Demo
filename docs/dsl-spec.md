# CPU 头文件 DSL 规范 v1.0

## 概述

CPU 头文件 (`.hdr`) 是一种声明式领域特定语言 (DSL)，用于描述 CPU 指令集的编码规则。编译器读取此文件后，能自动将原始二进制数据解码为汇编指令。

## 词法规则

### 注释
```
; 这是注释
; 注释以分号开头，持续到行尾
```

### 标识符
- 由字母、数字、下划线组成
- 不能以数字开头
- 区分大小写
- 示例: `nop`, `mov`, `dst_reg`, `imm_5`

### 数值
- **十六进制**: `0xE0`, `0xFF`, `0x20`
- **二进制**: `000`, `001`, `10101`
- **十进制**: `1`, `64`, `128`

### 关键字

| 关键字 | 用途 |
|--------|------|
| `arch` | 声明架构名称 |
| `endian` | 声明字节序 |
| `insn_size` | 声明指令长度策略 |
| `指令` | 声明一条指令的编码规则 |
| `字段` | 声明一个操作数字段的位范围与值映射 |

## 语句

### 1. 全局配置

```
arch "<名称>"
```
架构名称，用于标识和验证。

```
endian little|big
```
字节序：`little` (小端) 或 `big` (大端)。

```
insn_size fixed <N> | variable
```
指令长度策略：`fixed N` (所有指令固定 N 字节) 或 `variable` (每条指令在 `指令` 语句中用 `size=` 指定)。

### 2. 指令定义

```
指令 <助记符> mask=<掩码> pattern=<匹配值> [operands=<字段列表>] [size=<字节数>]
```

**参数：**
| 参数 | 必填 | 说明 |
|------|------|------|
| `mask` | 是 | 比特掩码(十六进制)，标记哪些位参与模式匹配 (1=参与) |
| `pattern` | 是 | 匹配值(十六进制)，在掩码覆盖的位上期望的值 |
| `operands` | 否 | 逗号分隔的字段名列表，声明操作数 |
| `size` | 否 | 指令字节数(仅变长模式需要) |

**匹配算法：**
```
if (input_word & mask) == pattern:
    匹配成功
```

**优先规则：**
1. 掩码位数多的优先（更精确的匹配优先）
2. 相同精度时，文件中先定义的优先

### 3. 字段定义

```
字段 <名称> bits=<高位>:<低位> [type=<类型>] [signed]
    <值名称> value=<比特值>
    <值名称> value=<比特值>
    ...
```

**参数：**
| 参数 | 必填 | 说明 |
|------|------|------|
| `bits` | 是 | 位范围，格式 `MSB:LSB` (高位:低位) |
| `type` | 否 | 字段类型：`immediate` (立即数)、`string` (字符串) |
| `signed` | 否 | 带符号的立即数 |

**值映射：**
- 缩进的行列出字段的每个可能取值
- `value` 为对应比特模式的值
- 未匹配的值直接以十六进制显示

### 4. 指令的 size 属性 (仅在变长模式下)

```
size=<N>          // N 字节指令
size=variable     // 变长指令 (如 print)
```

## 错误处理

- 未知字段名在 operands 中引用 → 解码时输出 `unknown_field`
- 字节无法匹配任何指令 → 输出 `db 0xNN`
- 头文件解析失败 → 编译器报错退出
